# Autodesk Package License File | Administrator Installation

Package licenses benefit Autodesk customers who have a maintenance plan or a subscription with multi-user access.

A package license gives [previous version rights](https://knowledge.autodesk.com/topics/account-management/software-downloads/previous-versions/subscription-previous-version-rights) to Autodesk maintenance plan customers and subscribers, allowing you to use eligible previous versions of Autodesk software. Beginning with 2018 products, package licenses extend to four previous versions. See [Download previous versions](/support/download-install/individuals/configure-install/download-previous-versions).

## Package licenses and previous version support

Package files include licenses for multiple releases of a product.

If you have a multi-user or maintenance plan for multiple versions of an Autodesk product, you can get a package file. Package files contain a set of licenses for previous releases of the same product, such as AutoCAD 2019-2021. Running an instance of any one of the releases uses one license in the package. A perpetual license that isn’t on a maintenance plan is called a "non-package license."

Package license files:

Authorize the use of the current version and limited previous versions of the same product.

Can be combined with license files for different products.

Can't be combined with non-package license files for product versions in the package. For example, you can't combine an AutoCAD 2021 package license file with a regular license file for AutoCAD 2016 or 2015.

Rely on both the PACKAGE and INCREMENT sections in the license file. When you combine license files, be sure to include both these sections.

Support license borrowing, license cascading, and options files.

Use the newer license file if an older file also exists.

### Package licenses for industry collections

Package licenses for industry collections allow every product in the industry collection to run with a shared license regardless of the release. For example, suppose you have a package license for the Product Design Collection. If so, you can run AutoCAD 2021 and AutoCAD 2020 at the same time (for multi-user only, one license consumed). And you can run AutoCAD 2021 and 3ds Max 2020 at the same time (for single or multi-user, one license consumed).

A license is borrowed for each product in a collection separately and consumes only one subscription from the server. For a suite, one license is borrowed that applies to all products in the same suite.

### How mixed licenses are consumed

If a license file contains a mix of different license types for the same product, a ranking logic determines which licenses from the pool are consumed first. Subscription licenses are consumed first, followed by perpetual licenses. Also, point products are consumed before collections that include the point product, which in turn are consumed before suites that include the point product. This logic optimizes the number of available licenses by assigning suite licenses only when no other licenses are available.

### Cascading sequences and license borrowing

License borrowing, options files, and license cascading work the same for package license files as they do for regular license files. When configuring options file parameters for a package license, use the [PACKAGE code](/support/download-install/admins/network-licenses/about-licensing-options-and-tools/configure-an-options-file) (for example, 64300ACD_T_F) and not the product/version feature code (for example, 87393ACD_2021_0F).
